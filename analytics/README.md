# Trust.xyz Privacy-First Analytics Stack

A production-ready, self-hosted analytics platform for trust.xyz with:
- **Zero cookies, zero personal data** — GDPR-compliant, privacy-by-design
- **Umami** — lightweight, cookieless web analytics
- **Dedicated crawler detection** — logs AI/ML bot activity separately (GPTBot, ClaudeBot, PerplexityBot, CCBot, etc.)
- **Easy deployment** — single command, under 30 minutes on any $6/month VPS

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│  trust.xyz (Static Site)                                │
│  ├─ <script src="/beacon.js"></script>                  │
│  └─ <script src="/crawler-beacon.js"></script>          │
└─────────────────────────────────────────────────────────┘
                           │
                ┌──────────┴──────────┐
                │                     │
        ┌───────▼────────┐   ┌────────▼────────┐
        │  Analytics      │   │  Crawler Log    │
        │  (Umami)        │   │  (Node.js)      │
        │  Port 3000      │   │  Port 3001      │
        └────────┬────────┘   └────────┬────────┘
                 │                     │
        ┌────────▼─────────────────────▼────────┐
        │  Nginx Reverse Proxy (HTTPS)          │
        │  analytics.trust.xyz → Let's Encrypt  │
        └───────────────────────────────────────┘
                           │
                ┌──────────┴──────────┐
                │                     │
        ┌───────▼────────┐   ┌────────▼────────┐
        │  PostgreSQL     │   │  NDJSON Logs    │
        │  (Docker)       │   │  crawler-hits   │
        └─────────────────┘   └─────────────────┘
```

## Quick Start

### Prerequisites
- Ubuntu 20.04+ or Debian 11+
- Root/sudo access
- A domain (e.g., `analytics.trust.xyz`) pointing to your server
- Minimum 512MB RAM, 2GB free disk space

### Deploy in 3 Commands

```bash
# 1. SSH to your server
ssh root@your-vps-ip

# 2. Clone and navigate to analytics directory
cd /path/to/analytics
sudo ./setup.sh

# 3. Complete SSL setup (follow the script's instructions)
sudo certbot certonly --standalone -d analytics.trust.xyz
sudo cp nginx.conf /etc/nginx/sites-available/analytics.trust.xyz
sudo ln -s /etc/nginx/sites-available/analytics.trust.xyz /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx
```

Done! Access your dashboard at `https://analytics.trust.xyz` (default login: `admin` / `umami`).

## File Overview

### `docker-compose.yml`
Defines Umami and PostgreSQL services. Key features:
- PostgreSQL 15-alpine for reliability and small footprint
- Umami with renamed tracker script (`beacon.js` instead of `umami.js`) to bypass ad-blockers
- Volume persistence for data survival
- Health checks for automatic recovery

### `.env.example`
All configuration variables:
- `DATABASE_URL` — PostgreSQL connection string
- `APP_SECRET` — security salt (auto-generated by setup.sh)
- `TRACKER_SCRIPT_NAME` — change from `umami.js` to avoid blockers
- `DISABLE_TELEMETRY` — prevents Umami from phoning home

### `nginx.conf`
Production-grade reverse proxy configuration:
- SSL/TLS with Let's Encrypt certificates
- Security headers (HSTS, CSP, X-Frame-Options, etc.)
- Gzip compression for tracking payload
- Rate limiting (10 req/s for UI, 50 req/s for tracking to handle crawler traffic)
- CORS for beacon requests

### `crawler-log/index.js`
Standalone Node.js HTTP server (no npm dependencies):
- Listens on port 3001
- Accepts POST requests to `/log` with page, referrer, UA, timestamp
- Detects 13+ known AI crawler patterns
- Logs crawler hits as newline-delimited JSON
- Returns 204 for humans, 200 for crawlers
- Auto-started via systemd service

**Detected crawlers:**
```
GPTBot, ClaudeBot, PerplexityBot, CCBot, 
Applebot-Extended, Googlebot-Extended, Bytespider, 
PetalBot, Diffbot, ImagesiftBot, YouBot, cohere-ai, 
anthropic-ai, HuggingFace, OpenAI
```

### `crawler-log/crawler-beacon.js`
Client-side script (~800 bytes minified) that:
- Fires on page load and visibility change
- Sends: page URL, referrer, user-agent, time-on-page
- Uses `navigator.sendBeacon()` for exit tracking
- Gracefully handles endpoint unavailability
- No console errors, no external dependencies

### `setup.sh`
Automated deployment script:
- Detects OS (Ubuntu/Debian)
- Installs Docker and Docker Compose
- Generates `.env` with random secrets
- Starts containers
- Creates systemd service for crawler logger
- Prints next-steps checklist

## Embedding Analytics in Your Site

### 1. Umami Tracking Script
Add to every page `<head>` (or end of `<body>`):

```html
<script async
  src="https://analytics.trust.xyz/beacon.js"
  data-website-id="YOUR_WEBSITE_ID">
</script>
```

Find `YOUR_WEBSITE_ID` in the Umami dashboard after creating a website.

### 2. Crawler Detection Script
Add to every page (can be in `<head>` or `<body>`):

```html
<script src="https://analytics.trust.xyz/your-domain/crawler-beacon.js"></script>
```

The crawler-beacon.js will:
- Load on every page
- Detect if the visitor is an AI crawler
- POST to the local `/log` endpoint
- Log results to `crawler-hits.ndjson`

## Querying Crawler Data

The crawler detection logs are in `crawler-log/crawler-hits.ndjson` (newline-delimited JSON).

### View recent crawler hits:
```bash
tail -f crawler-log/crawler-hits.ndjson
```

### Pretty-print with jq:
```bash
cat crawler-log/crawler-hits.ndjson | jq '.'
```

### Count crawlers by type:
```bash
cat crawler-log/crawler-hits.ndjson | jq -r '.ua' | sort | uniq -c | sort -rn
```

### Filter for specific crawler:
```bash
cat crawler-log/crawler-hits.ndjson | jq 'select(.ua | contains("ClaudeBot"))'
```

### Get crawlers per day:
```bash
cat crawler-log/crawler-hits.ndjson | jq -r '.ts[:10]' | sort | uniq -c
```

## Operating the Stack

### Check service status
```bash
# Umami and PostgreSQL
docker compose ps

# Crawler logger
systemctl status trust-xyz-crawler-log
```

### View logs
```bash
# Umami application
docker compose logs -f umami

# PostgreSQL
docker compose logs -f postgres

# Crawler logger
journalctl -u trust-xyz-crawler-log -f
```

### Restart services
```bash
# All services
docker compose restart

# Crawler logger only
systemctl restart trust-xyz-crawler-log
```

### Stop services (gracefully)
```bash
docker compose stop
systemctl stop trust-xyz-crawler-log
```

### Backup your database
```bash
docker compose exec postgres pg_dump -U umami umami > backup-$(date +%Y%m%d).sql
```

### Restore from backup
```bash
docker compose exec -T postgres psql -U umami umami < backup-YYYYMMDD.sql
```

## Troubleshooting

### "Database connection refused"
- Wait 30 seconds for PostgreSQL to start: `docker compose logs postgres`
- Verify `DATABASE_URL` in `.env`
- Check `.env` vs `docker-compose.yml` values match

### "Port 3000 already in use"
```bash
sudo lsof -i :3000
kill <PID>
docker compose restart umami
```

### Tracker script not loading
- Check nginx logs: `sudo tail -f /var/log/nginx/analytics.trust.xyz.error.log`
- Verify certificate is valid: `sudo certbot renew --dry-run`
- Confirm `TRACKER_SCRIPT_NAME` in `.env` matches the `<script src=">` in your HTML

### Crawler logger not receiving requests
- Verify port 3001 is open and service is running: `systemctl status trust-xyz-crawler-log`
- Check firewall: `sudo ufw status` (allow port 3001 if needed)
- Review logs: `journalctl -u trust-xyz-crawler-log -n 50`
- Test endpoint: `curl -X POST http://localhost:3001/log -H 'Content-Type: application/json' -d '{"ua":"ClaudeBot"}'`

### SSL certificate not renewing
```bash
sudo certbot renew --force-renewal
sudo systemctl restart nginx
```

### High memory usage
- Umami is lightweight (~150MB typical)
- PostgreSQL grows with data; consider archiving old logs
- Check: `docker stats`

## Security Considerations

### Network
- Umami listens only on `127.0.0.1:3000` (not public)
- Crawler logger listens only on `127.0.0.1:3001` (not public)
- Nginx acts as sole entry point with SSL/TLS
- Only expose `analytics.trust.xyz` publicly

### Credentials
- Change Umami admin password after first login
- Store `.env` securely (not in version control)
- Regenerate `APP_SECRET` and `DB_PASSWORD` annually

### Data
- Database lives in Docker volumes (check backups)
- NDJSON logs are plain text; consider rotating old logs
- All communication over HTTPS after SSL setup

### Rate Limiting
- Nginx limits: 10 req/s for UI, 50 req/s for tracking
- Prevents DoS and abuse
- Adjust in `nginx.conf` if needed

## Customization

### Change tracker script name (to bypass more ad-blockers)
Edit `.env`:
```
TRACKER_SCRIPT_NAME=metrics
```
Then: `docker compose restart umami`
Update HTML: `<script src="https://analytics.trust.xyz/metrics.js"></script>`

### Add more crawler patterns
Edit `crawler-log/index.js`, update `AI_CRAWLERS` array, restart service:
```bash
systemctl restart trust-xyz-crawler-log
```

### Adjust rate limiting
Edit `nginx.conf`, update `limit_req_zone` rates, test, reload nginx:
```bash
sudo nginx -t && sudo systemctl reload nginx
```

## Maintenance

### Monthly
- Review crawler activity: `wc -l crawler-log/crawler-hits.ndjson`
- Check disk space: `df -h /var`
- Verify backups are working

### Quarterly
- Update Docker images: `docker compose pull && docker compose up -d`
- Review Umami for insights and anomalies
- Archive old crawler logs: `gzip crawler-log/crawler-hits.ndjson.* && mv *.gz archive/`

### Annually
- Regenerate APP_SECRET and DB_PASSWORD
- Rotate SSL certificates (automatic with certbot, but verify)
- Audit which crawlers are accessing your site
