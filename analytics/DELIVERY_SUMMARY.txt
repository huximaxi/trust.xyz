DEPLOYMENT TIMELINE
================================================================================

Estimated time to production: 30 minutes

Phase 1: Automated setup          (10 min)
  $ sudo ./setup.sh
  → Docker + Compose installed
  → .env with random secrets
  → Containers running
  → Crawler logger systemd service created

Phase 2: SSL certificate          (5 min)
  $ sudo certbot certonly --standalone -d analytics.trust.xyz

Phase 3: Nginx proxy              (5 min)
  Copy nginx.conf → /etc/nginx/sites-available/
  Enable site → /etc/nginx/sites-enabled/
  $ sudo nginx -t && restart

Phase 4: DNS & initial access     (2 min)
  Point DNS A record to server
  Verify with curl & browser

Phase 5: Dashboard configuration  (3 min)
  Change admin password
  Create website profile
  Get tracking ID

Phase 6: HTML integration         (2 min)
  Embed tracking scripts
  Test with curl

Phase 7: Monitoring               (2 min)
  Run monitor.sh
  Create daily backups (optional)

Phase 8: Post-deployment check    (1 min)
  Verify services, test tracking, check logs

CUSTOMIZATION & EXTENSIONS
================================================================================

Easy Customizations
  • Change tracker script name (avoid ad-blockers): TRACKER_SCRIPT_NAME in .env
  • Adjust rate limits: Edit nginx.conf limit_req_zone values
  • Add more crawler patterns: Edit AI_CRAWLERS array in index.js
  • Change database password: Edit DB_PASSWORD in .env

Advanced Extensions
  • Add Prometheus metrics exporter (monitoring)
  • Integrate with Slack/webhook alerts
  • Set up S3 backup automation
  • Add Redis cache layer (scaling)
  • Create Kafka topic for streaming events

Not Recommended
  • Don't expose Umami or PostgreSQL publicly
  • Don't skip SSL/TLS encryption
  • Don't commit .env to version control
  • Don't change Umami admin password via UI (use env vars)
