QUICK START
================================================================================

1. Deploy:
   $ sudo ./setup.sh
   
2. Configure SSL:
   $ sudo certbot certonly --standalone -d analytics.trust.xyz
   $ sudo cp nginx.conf /etc/nginx/sites-available/analytics.trust.xyz
   $ sudo ln -s /etc/nginx/sites-available/analytics.trust.xyz /etc/nginx/sites-enabled/
   $ sudo nginx -t && sudo systemctl restart nginx

3. Access dashboard:
   https://analytics.trust.xyz (default: admin / umami)

4. Embed in pages:
   <script async src="https://analytics.trust.xyz/beacon.js" 
           data-website-id="YOUR_WEBSITE_ID"></script>
   <script src="https://analytics.trust.xyz/your-domain/crawler-beacon.js"></script>

5. Monitor:
   $ ./monitor.sh              (Health check)
   $ tail -f crawler-log/crawler-hits.ndjson  (Crawler activity)

nginx.conf (3.6 KB)
───────────────────
Production reverse proxy for analytics.trust.xyz:

HTTP → HTTPS Redirect
  • Listens on port 80
  • Redirects all requests to HTTPS
  • Serves .well-known/acme-challenge for certbot

HTTPS Server (port 443)
  • SSL/TLS with Let's Encrypt certificates
  • TLS 1.2 + 1.3 only
  • Security headers: HSTS, CSP, X-Frame-Options, etc.
  • Gzip compression enabled

Rate Limiting
  • Analytics UI: 10 req/s (burst 20)
  • Tracking endpoint: 50 req/s (burst 100)
  • Prevents DoS and abuse

Proxying
  • /api/collect         → Tracking requests (high limit)
  • /                    → Umami app (rate limited)
  • Security: denies dot-files, backups, sensitive paths

Logging
  • access.log           HTTP request log
  • error.log            Error messages

index.js in crawler-log/ (4.6 KB, executable)
──────────────────────────────────────────────
Standalone Node.js HTTP server for crawler detection:

Purpose: Accept page load events from crawler-beacon.js
         and log AI crawler activity to NDJSON

Features:
  • Pure Node.js (no npm, no frameworks)
  • HTTP (not HTTPS; behind nginx in production)
  • CORS support for cross-origin requests
  • Payload size limit (10 KB DoS protection)
  • Graceful shutdown (SIGTERM/SIGINT)

Endpoints:
  POST /log
    Body: { page, referrer, ua, ts }
    Returns: 200 (crawler logged) or 204 (human)
    
  OPTIONS /
    CORS preflight handler
    Returns: 204 with CORS headers

Detected Crawlers (13+):
  GPTBot, ClaudeBot, PerplexityBot, CCBot,
  Applebot-Extended, Googlebot-Extended, Bytespider,
  PetalBot, Diffbot, ImagesiftBot, YouBot,
  cohere-ai, anthropic-ai, HuggingFace, OpenAI

Output: crawler-hits.ndjson (append-only)
  { ts, page, referrer, ua, ip }

Port: 3001 (internal only)
Started by: systemd service trust-xyz-crawler-log

crawler-beacon.js in crawler-log/ (1.9 KB)
────────────────────────────────────────────
Browser JavaScript library for crawler detection:

Purpose: Detect AI crawlers and send events to /log endpoint

Features:
  • Auto-executes on page load (IIFE)
  • No external dependencies
  • No npm, no build step needed
  • Error handling (silent fail, no console noise)
  • Minimal footprint (~800 bytes minified)

Events Sent:
  1. Page load       — Initial beacon
  2. Visibility change — Tab hidden/shown
  3. Unload          — User leaves page
  
Payload:
  {
    page: window.location.href,
    referrer: document.referrer,
    ua: navigator.userAgent,
    ts: ISO 8601 timestamp,
    timeOnPage: seconds since load
  }

Transport:
  • Prefers navigator.sendBeacon() (no-blocking exit ping)
  • Falls back to fetch() if available
  • Silently fails if endpoint unavailable (no console errors)

CORS: Origin check in index.js (allows https://trust.xyz)

crawler-hits.ndjson
────────────────────
Output log file (append-only, newline-delimited JSON):

Created by: index.js server on first /log POST request
Location: crawler-log/crawler-hits.ndjson
Format: One JSON object per line (NDJSON)

Example lines:
  {"ts":"2026-04-15T10:23:45.123Z","page":"https://trust.xyz/","referrer":"direct","ua":"ClaudeBot","ip":"203.0.113.42"}
  {"ts":"2026-04-15T10:24:12.456Z","page":"https://trust.xyz/about","referrer":"google.com","ua":"GPTBot","ip":"198.51.100.5"}

Fields:
  ts         ISO 8601 timestamp of event
  page       Full URL visited by crawler
  referrer   HTTP Referer header (or 'direct')
  ua         User-Agent string
  ip         Client IP (from X-Forwarded-For or socket)

Querying with jq:
  cat crawler-hits.ndjson | jq '.'         # Pretty-print
  cat crawler-hits.ndjson | jq '.ua' | sort | uniq -c   # Crawler counts
  grep "2026-04-15" crawler-hits.ndjson | wc -l         # Daily totals

Rotation: Archive quarterly (gzip and move to backup)

================================================================================

  2. Install Docker & Docker Compose
  3. Create directory structure
  4. Generate .env with random secrets

  1. certbot --standalone
  2. Copy nginx.conf to /etc/nginx/sites-available/

  4. nginx -t && restart nginx
  5. Point DNS to server
  6. Access dashboard, create website, get tracking ID
  7. Embed tracking scripts in HTML pages

Result →
  ✓ Umami dashboard at https://analytics.trust.xyz
  ✓ Tracking script on all pages
  ✓ Crawler logger running

================================================================================
SECURITY MODEL
================================================================================

No Cookies:
  ✓ Umami uses device fingerprinting (not cookies)
  ✓ No GDPR "cookie consent" popup needed
  ✓ No PII collection

No Personal Data:
  ✓ IP addresses are hashed, never stored raw
  ✓ No user IDs, emails, or identifiers
  ✓ Anonymous aggregation only

Encryption:
  ✓ All public traffic over HTTPS (nginx + Let's Encrypt)
  ✓ Database password in .env (not in code)
  ✓ APP_SECRET for session hashing (random, per deployment)

Isolation:
  ✓ Umami (port 3000) only accessible via nginx
  ✓ PostgreSQL (port 5432) not exposed publicly
  ✓ Crawler logger (port 3001) not exposed publicly
  ✓ Docker volumes owned by container user (not root)

Rate Limiting:
  ✓ Nginx enforces 10-50 req/s per IP
  ✓ Prevents DoS and abuse
  ✓ Crawler logger validates JSON payloads

Logging:
  ✓ Access logs in /var/log/nginx/ (rotate with logrotate)
  ✓ Error logs for troubleshooting
  ✓ Crawler hits in NDJSON (plaintext, consider access controls)
